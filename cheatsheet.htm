Background

Follow this checklist when creating a website project.

Why?

Although version control may seem overkill for a small website project it is a good practise to get into these habits.

Use these projects to internalise the habits so for larger projects in the future you won't have to think about them.

Checklist

  • Open bash terminal and cd to your projects directory.
  • Use mkdir to create a directory for your project
  • cd into your new project directory and mkdir a css and images directory as necessary
  • git init the new project directory
  • Create and name a new repository on GitHub
  • echo "# new project" >> README.md
  • git add README.md
  • git commit -m "first commit"
  • git remote add origin https://github.com/yourname/your-new-project.git
  • git push -u origin master
  • Create your htm file.
  • Create your css file and save to the css folder
  • Periodically add and commit files to git
  • git add .
  • git commit -m "describe your changes"
  • git push
